## 概要

本文件說明商戶方如何透過商戶整合 API 將 COMBI 3 遊戲接入商戶方系統。
該 API 允許商戶方執行以下操作：

- 建立玩家帳號
- 對玩家錢包進行轉帳
- 全額轉出玩家餘額
- 查詢下注紀錄
- 查詢轉款紀錄
- 以已驗證的玩家 Session 啟動遊戲（支援 Legacy 與 Callback 模式）
- 取得並設定下注限額與籌碼面額

所有 API 端點均使用 **HTTP POST** 方法，請求與回應的內容均為 **JSON** 格式。

## 對接申請與開通

### 開戶前，商戶須提供資訊

- 幣別代碼
- 各玩法預設投注限額設定
  - maxBetLimit (總限額: 所有玩法均不可大於此值)
  - minBetLimit (總限額: 所有玩法均不可小於此值)
  - bigSmall
    - maxBetLimit
    - minBetLimit
  - oddEven
    - maxBetLimit
    - minBetLimit
  - eachDouble
    - maxBetLimit
    - minBetLimit
  - eachTripple
    - maxBetLimit
    - minBetLimit
  - sum
    - maxBetLimit
    - minBetLimit
  - single
    - maxBetLimit
    - minBetLimit
  - anyTripple
    - maxBetLimit
    - minBetLimit
- Callback Url
  - LoginPlayer: 例如 https://merchant.example.com/login-player
  - UpdateBalance: 例如 https://merchant.example.com/update-balance
- IP 白名單列表: 所有呼叫對接 API 的來源服務器 IP，以便遊戲方開通

### 開戶後，遊戲方會提供給商戶的對接資訊

- 商戶 Id (Merchant Id)
- 哈希金鑰 (Hash Key)
- IP 白名單列表: 所有呼叫商戶所提供 Callback Urls 的來源服務器 IP，請商戶開通後通知

## 驗證機制

每個 API 請求都必須通過以下方式進行身分驗證：

1. **商家 ID（merchantId）**：上線申請時系統分配給您的唯一商家識別碼
2. **時戳（timestamp）**：當前的 Unix 時戳，單位為秒（10 位數）
3. **簽章（hash）**：將請求參數與您的密鑰拼接後，經 SHA256 演算法生成的雜湊值

### 時戳驗證

- 時戳必須在伺服器當前時間的 **5〜10 秒** 範圍內
- 超出此範圍的時戳將被拒絕，並傳回錯誤代碼 `1002`

### IP 白名單

- 僅允許商戶方所提供並設置在遊戲方白名單中的 IP 發送請求到商戶整合 API，不在白名單中的 IP 將被拒絕，傳回錯誤代碼 `1005`
- 同時商戶方也需要將遊戲方的 IP 加入白名單，否則遊戲方可能將無法正常使用商戶方的 Callback API (如：LoginPlayer, UpdateBalance)

## 簽章生成

### 雜湊密鑰

您的雜湊密鑰為上線申請時系統提供的一個 **32 位元 BASE64** 字串。請妥善保管此密鑰，絕不得暴露於客戶端程式碼中。

### 簽章演算法

```
hash = SHA256(param1 + "&" + param2 + "&" + ... + "&" + hashKey)
```

**步驟：**

1. 依照各 API 指定的順序，將所有參數以 `&` 作為分隔符拼接
2. 在末尾追加 `&`，接著拼接您的 `hashKey`
3. 對拼接後的字串計算 SHA256 雜湊值
4. 將結果轉換為小寫的十六進制字串（共 64 個字元）

### 範例（JavaScript / Node.js）

```javascript
const crypto = require('crypto');

function generateSignature(params, hashKey) {
  const data = params.join('&') + '&' + hashKey;
  return crypto.createHash('sha256').update(data).digest('hex');
}

// AccountCreate 範例
const merchantId = 'MERCHANT001';
const account = 'player123';
const timestamp = Math.floor(Date.now() / 1000);
const hashKey = 'your-32-char-secret-key-here!!!';

const hash = generateSignature([merchantId, account, timestamp.toString()], hashKey);
```

### 範例（Python）

```python
import hashlib
import time

def generate_signature(params, hash_key):
    data = '&'.join(params) + '&' + hash_key
    return hashlib.sha256(data.encode()).hexdigest()

# AccountCreate 範例
merchant_id = 'MERCHANT001'
account = 'player123'
timestamp = int(time.time())
hash_key = 'your-32-char-secret-key-here!!!'

hash_value = generate_signature([merchant_id, account, str(timestamp)], hash_key)
```

### 範例（Java）

```java
import java.security.MessageDigest;
import java.nio.charset.StandardCharsets;

public String generateSignature(String[] params, String hashKey) {
    String data = String.join("&", params) + "&" + hashKey;
    MessageDigest digest = MessageDigest.getInstance("SHA-256");
    byte[] hash = digest.digest(data.getBytes(StandardCharsets.UTF_8));
    StringBuilder hexString = new StringBuilder();
    for (byte b : hash) {
        String hex = Integer.toHexString(0xff & b);
        if (hex.length() == 1) hexString.append('0');
        hexString.append(hex);
    }
    return hexString.toString();
}
```

## 通用回應格式

所有 API 的回應均遵循以下結構：

```json
{
  "success": true | false,
  "errorCode": 0,
  "errorMessage": "",
  "data": { ... } | null
}
```

| 欄位         | 類型          | 說明                                     |
| ------------ | ------------- | ---------------------------------------- |
| success      | boolean       | 請求成功為 `true`，失敗為 `false`    |
| errorCode    | integer       | 成功時為 `0`，失敗時傳回對應的錯誤代碼 |
| errorMessage | string        | 成功時為空字串，失敗時傳回錯誤說明       |
| data         | object / null | 成功時傳回回應資料，失敗時為 `null`    |

## API 端點

### 建立帳號

在遊戲系統中建立一個新的玩家帳號。

**端點：** `POST /integration/account/create`

#### 請求

| 參數       | 類型    | 必填 | 說明                           |
| ---------- | ------- | ---- | ------------------------------ |
| merchantId | string  | 是   | 您的商家 ID                    |
| account    | string  | 是   | 唯一的玩家帳號識別碼           |
| timestamp  | integer | 是   | Unix 時戳，單位為秒（10 位數） |
| hash       | string  | 是   | 請求簽章                       |

**簽章參數（依序）：**

```
hash = SHA256(merchantId + "&" + account + "&" + timestamp + "&" + hashKey)
```

#### 請求範例

```json
{
  "merchantId": "MERCHANT001",
  "account": "player123",
  "timestamp": 1706886400,
  "hash": "a1b2c3d4e5f6..."
}
```

#### 回應

**成功：**

```json
{
  "success": true,
  "errorCode": 0,
  "errorMessage": "",
  "data": null
}
```

**失敗（帳號已存在）：**

```json
{
  "success": false,
  "errorCode": 2001,
  "errorMessage": "Account already exists",
  "data": null
}
```

### 轉帳

對玩家的遊戲錢包進行轉帳操作。

**端點：** `POST /integration/transfer`

#### 請求

| 參數       | 類型    | 必填 | 說明                                               |
| ---------- | ------- | ---- | -------------------------------------------------- |
| merchantId | string  | 是   | 您的商家 ID                                        |
| account    | string  | 是   | 玩家帳號識別碼                                     |
| transferId | string  | 是   | 唯一的轉款 ID（用於冪等性保障）                    |
| type       | integer | 是   | 0 = 轉入（轉入遊戲），1 = 轉出（從遊戲提出至商家） |
| amount     | number  | 是   | 轉款金額（必須 > 0）                               |
| timestamp  | integer | 是   | Unix 時戳，單位為秒                                |
| hash       | string  | 是   | 請求簽章                                           |

**簽章參數（依序）：**

```
hash = SHA256(merchantId + "&" + account + "&" + type + "&" + amount + "&" + timestamp + "&" + hashKey)
```

#### 請求範例（轉入）

```json
{
  "merchantId": "MERCHANT001",
  "account": "player123",
  "transferId": "TXN20260202001",
  "type": 0,
  "amount": 100.00,
  "timestamp": 1706886400,
  "hash": "a1b2c3d4e5f6..."
}
```

#### 請求範例（轉出）

```json
{
  "merchantId": "MERCHANT001",
  "account": "player123",
  "transferId": "TXN20260202002",
  "type": 1,
  "amount": 50.00,
  "timestamp": 1706886400,
  "hash": "a1b2c3d4e5f6..."
}
```

#### 回應

**成功：**

```json
{
  "success": true,
  "errorCode": 0,
  "errorMessage": "",
  "data": {
    "balance": 150.00
  }
}
```

| 欄位    | 類型   | 說明                 |
| ------- | ------ | -------------------- |
| balance | number | 轉款後玩家的最新餘額 |

**失敗（餘額不足）：**

```json
{
  "success": false,
  "errorCode": 3001,
  "errorMessage": "Insufficient balance",
  "data": null
}
```

> **注意：** 轉出（type=1）成功後，系統會自動關閉該玩家的活動 Launch Session。

```
hash = SHA256(merchantId + "&" + formattedStartTime + "&" + pageSize + "&" + pageNumber + "&" + timestamp + "&" + hashKey)
```

## 回調機制

當商家啟用 Callback 模式（`callbackEnabled = true`）時，系統會在特定事件發生時主動回調商家的伺服器。

### 前置設定

商家須在系統中配置以下兩個 URL：

| 設定                         | 說明                         |
| ---------------------------- | ---------------------------- |
| `loginPlayerCallbackUrl`   | 玩家登入驗證時的回調 URL     |
| `updateBalanceCallbackUrl` | 玩家離線後餘額同步的回調 URL |

### 玩家登入驗證

**觸發時機：** 玩家開啟遊戲後，前端自動呼叫 `POST /integration/launch/session/start`，系統向商家發送 LoginPlayer 回調以驗證玩家身分。

**回調請求（系統 → 商家）：**

```
POST {loginPlayerCallbackUrl}
```

```json
{
  "merchantId": "MERCHANT001",
  "playerId": "pid-001",
  "account": "player123",
  "accessToken": "merchantToken001",
  "currency": "USDT",
  "timestamp": 1706886400,
  "hash": "a1b2c3d4e5f6..."
}
```

| 欄位        | 說明                                                           |
| ----------- | -------------------------------------------------------------- |
| merchantId  | 商家 ID                                                        |
| playerId    | 啟動遊戲時提供的商家端玩家 ID                                  |
| account     | 玩家帳號                                                       |
| accessToken | 啟動遊戲時提供的商家端 Access Token                            |
| currency    | 幣別                                                           |
| timestamp   | Unix 時戳                                                      |
| hash        | 簽章：`SHA256(merchantId + "&" + timestamp + "&" + hashKey)` |

**商家須回傳：**

```json
{
  "success": true
}
```

- 回傳 `{ "success": true }` 表示驗證通過，玩家可正常進入遊戲
- 回傳其他結果或 HTTP 非 200 狀態碼表示驗證失敗，玩家無法進入遊戲

**重試機制：** 最多重試 2 次（共 3 次嘗試），單次超時 5 秒。

### 通知商戶轉回玩家餘額

**觸發時機：** 玩家離線後，系統自動發送 UpdateBalance 回調通知商家轉回餘額。

**回調請求（系統 → 商家）：**

```
POST {updateBalanceCallbackUrl}
```

```json
{
  "merchantId": "MERCHANT001",
  "playerId": "pid-001",
  "account": "player123",
  "accessToken": "merchantToken001",
  "currency": "USDT",
  "timestamp": 1706886400,
  "hash": "a1b2c3d4e5f6..."
}
```

**欄位說明與 LoginPlayer 回調相同。**

**商家須回傳：**

```json
{
  "success": true
}
```

**重試機制：** 最多重試 2 次（共 3 次嘗試），單次超時 5 秒。

### 回調流程圖

```
商家呼叫 /integration/launch (Callback 模式)
     │ 提供 playerId + accessToken
     ▼
系統建立 LaunchSession，回傳遊戲 URL
     │
     ▼
玩家開啟遊戲 URL
     │
     ▼
前端呼叫 POST /integration/launch/session/start
     │
     ▼
系統發送 LoginPlayer 回調 → 商家伺服器
     │ 商家回傳 { success: true }
     ▼
玩家正常遊戲中...
     │
     ▼
玩家關閉遊戲（WebSocket 斷線）
     │
     ⏰ 等待 30 秒寬限期
     │
     ▼
系統發送 UpdateBalance 回調 → 商家伺服器
     │ 商家同步玩家餘額
     ▼
Launch Session 結束
```

## 資料類型

### 數字投注類型

| 類型       | 說明                                        | 是否需要選項值               |
| ---------- | ------------------------------------------- | ---------------------------- |
| SMALL      | 各位數字之和為 0〜13                        | 否                           |
| BIG        | 各位數字之和為 14〜27                       | 否                           |
| ODD        | 各位數字之和為奇數                          | 否                           |
| EVEN       | 各位數字之和為偶數                          | 否                           |
| ANY_TRIPLE | 3 位數字全部相同                            | 否                           |
| DOUBLE     | 指定的雙重組合（例如 `"00"`、`"11"`）   | 是（`"00"` 〜 `"99"`）   |
| TRIPLE     | 指定的三重組合（例如 `"000"`、`"111"`） | 是（`"000"` 〜 `"999"`） |
| SUM        | 指定的數字總和                              | 是（`"3"` 〜 `"27"`）    |
| SINGLE     | 指定的單個數字至少出現一次                  | 是（`"0"` 〜 `"9"`）     |

### 下注結果類型

| 結果    | 說明                         |
| ------- | ---------------------------- |
| PENDING | 投注進行中，回合尚未結束     |
| WIN     | 玩家獲勝                     |
| LOSE    | 玩家輸掉                     |
| REFUND  | 投注已退款（回合取消或平局） |

### 投注類型

| 類型  | 說明                                 |
| ----- | ------------------------------------ |
| DIGIT | 數字投注，預測價格最後三位數字的特徵 |

### Launch Session 狀態

| 狀態       | 說明                         |
| ---------- | ---------------------------- |
| ACTIVE     | 活動中                       |
| SUPERSEDED | 被新的 Session 取代          |
| CLOSED     | 已關閉（轉款或全額轉出觸發） |
| EXPIRED    | 已過期                       |

## 程式碼範例

### 完整整合範例（Node.js）

```javascript
const crypto = require('crypto');
const axios = require('axios');

class GameIntegration {
  constructor(merchantId, hashKey, baseUrl) {
    this.merchantId = merchantId;
    this.hashKey = hashKey;
    this.baseUrl = baseUrl;
  }

  generateSignature(params) {
    const data = params.join('&') + '&' + this.hashKey;
    return crypto.createHash('sha256').update(data).digest('hex');
  }

  getTimestamp() {
    return Math.floor(Date.now() / 1000);
  }

  formatDateForSignature(date) {
    const pad = (n, len = 2) => n.toString().padStart(len, '0');
    return (
      date.getUTCFullYear().toString() +
      pad(date.getUTCMonth() + 1) +
      pad(date.getUTCDate()) +
      pad(date.getUTCHours()) +
      pad(date.getUTCMinutes()) +
      pad(date.getUTCSeconds()) +
      pad(date.getUTCMilliseconds(), 3)
    );
  }

  async createAccount(account) {
    const timestamp = this.getTimestamp();
    const hash = this.generateSignature([
      this.merchantId, account, timestamp.toString(),
    ]);
    const response = await axios.post(`${this.baseUrl}/integration/account/create`, {
      merchantId: this.merchantId, account, timestamp, hash,
    });
    return response.data;
  }

  async transfer(account, transferId, type, amount) {
    const timestamp = this.getTimestamp();
    const hash = this.generateSignature([
      this.merchantId, account, type.toString(), amount.toString(), timestamp.toString(),
    ]);
    const response = await axios.post(`${this.baseUrl}/integration/transfer`, {
      merchantId: this.merchantId, account, transferId, type, amount, timestamp, hash,
    });
    return response.data;
  }

  async allTransferOut(account, transferId) {
    const timestamp = this.getTimestamp();
    const hash = this.generateSignature([
      this.merchantId, account, timestamp.toString(),
    ]);
    const response = await axios.post(`${this.baseUrl}/integration/all-transfer-out`, {
      merchantId: this.merchantId, account, transferId, timestamp, hash,
    });
    return response.data;
  }

  async launchGame(account, options = {}) {
    const timestamp = this.getTimestamp();
    const hash = this.generateSignature([
      this.merchantId, account, timestamp.toString(),
    ]);
    const body = { merchantId: this.merchantId, account, timestamp, hash };
    if (options.playerId) body.playerId = options.playerId;
    if (options.accessToken) body.accessToken = options.accessToken;
    if (options.betLimits) body.betLimits = options.betLimits;
    const response = await axios.post(`${this.baseUrl}/integration/launch`, body);
    return response.data;
  }

  async getBetHistory(startBetTime, pageSize, pageNumber) {
    const timestamp = this.getTimestamp();
    const formattedTime = this.formatDateForSignature(new Date(startBetTime));
    const hash = this.generateSignature([
      this.merchantId, formattedTime, pageSize.toString(),
      pageNumber.toString(), timestamp.toString(),
    ]);
    const response = await axios.post(`${this.baseUrl}/integration/bets`, {
      merchantId: this.merchantId, startBetTime, pageSize, pageNumber, timestamp, hash,
    });
    return response.data;
  }

  async getTransferHistory(startTime, pageSize, pageNumber) {
    const timestamp = this.getTimestamp();
    const formattedTime = this.formatDateForSignature(new Date(startTime));
    const hash = this.generateSignature([
      this.merchantId, formattedTime, pageSize.toString(),
      pageNumber.toString(), timestamp.toString(),
    ]);
    const response = await axios.post(`${this.baseUrl}/integration/transfers`, {
      merchantId: this.merchantId, startTime, pageSize, pageNumber, timestamp, hash,
    });
    return response.data;
  }

  async getBetLimit() {
    const timestamp = this.getTimestamp();
    const hash = this.generateSignature([this.merchantId, timestamp.toString()]);
    const response = await axios.post(`${this.baseUrl}/integration/config/bet-limit/get`, {
      merchantId: this.merchantId, timestamp, hash,
    });
    return response.data;
  }

  async setBetLimit(minBetAmount, maxBetAmount) {
    const timestamp = this.getTimestamp();
    const hash = this.generateSignature([
      this.merchantId, minBetAmount.toString(), maxBetAmount.toString(), timestamp.toString(),
    ]);
    const response = await axios.post(`${this.baseUrl}/integration/config/bet-limit`, {
      merchantId: this.merchantId, minBetAmount, maxBetAmount, timestamp, hash,
    });
    return response.data;
  }

  async getTokenValues() {
    const timestamp = this.getTimestamp();
    const hash = this.generateSignature([this.merchantId, timestamp.toString()]);
    const response = await axios.post(`${this.baseUrl}/integration/config/token-values/get`, {
      merchantId: this.merchantId, timestamp, hash,
    });
    return response.data;
  }

  async setTokenValues(tokenValues) {
    const timestamp = this.getTimestamp();
    const tokenValuesCSV = tokenValues.join(',');
    const hash = this.generateSignature([
      this.merchantId, tokenValuesCSV, timestamp.toString(),
    ]);
    const response = await axios.post(`${this.baseUrl}/integration/config/token-values`, {
      merchantId: this.merchantId, tokenValues, timestamp, hash,
    });
    return response.data;
  }
}

// 使用範例
async function main() {
  const integration = new GameIntegration(
    'MERCHANT001',
    'your-32-char-secret-key-here!!!',
    'https://api.your-game-domain.com'
  );

  // 建立帳號
  const createResult = await integration.createAccount('player123');
  console.log('建立帳號:', createResult);

  // 轉入 100 USDT
  const depositResult = await integration.transfer('player123', 'DEP-' + Date.now(), 0, 100);
  console.log('轉入:', depositResult);

  // 啟動遊戲（Legacy 模式）
  const launchResult = await integration.launchGame('player123');
  console.log('遊戲 URL:', launchResult.data?.url);

  // 啟動遊戲（Callback 模式 + betLimits）
  const callbackLaunch = await integration.launchGame('player123', {
    playerId: 'pid-001',
    accessToken: 'merchantToken001',
    betLimits: { bigSmall: 500, oddEven: 500 },
  });
  console.log('Callback 遊戲 URL:', callbackLaunch.data?.url);

  // 取得下注紀錄
  const betsResult = await integration.getBetHistory('2026-01-01T00:00:00.000Z', 50, 1);
  console.log('下注紀錄:', betsResult);

  // 轉出 50 USDT
  const withdrawResult = await integration.transfer('player123', 'WDR-' + Date.now(), 1, 50);
```
